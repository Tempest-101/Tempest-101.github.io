剑指Offer第二版刷题追踪

算法小白第一次系统的刷题经历。

offer 03.数组中重复的数字

在一个长度为 n 的数组 nums 里的所有数字都在 0～n-1 的范围内。数组中某些数字是重复的，但不知道有几个数字重复了，也不知道每个数字重复了几次。请找出数组中任意一个重复的数字。

接触此题第一反应是将数组排序，然后遍历一遍，同时与前者或后者进行比较，只要相等就返回遍历到的数值。再进一步想，何不在排序时就进行判断呢？由此我运用了插入排序的方法，这样就能在排序时进行重复数的查找，只要排序进行到第一个重复数，就能立刻返回得出答案。

优化

本题有条件范围0~n-1，因此我们考虑运用下标法，具体逻辑为遍历数组，使得数组的索引与值相等。而达到这一目的一定要使用交换操作，在交换时若发现交换的两个元素相等，则直接返回该元素。

offer 04.二维数组中的查找

在一个 n * m 的二维数组中，每一行都按照从左到右递增的顺序排序，每一列都按照从上到下递增的顺序排序。请完成一个高效的函数，输入这样的一个二维数组和一个整数，判断数组中是否含有该整数。

由该数组的性质可知，一个数的左侧与上侧的所有数一定比它本身要小，因此我将起始点置于右上角，这样当target比所在点大时，那左侧一排的所有数都可以排除，下一个探查点只能是下方；而如果比所在点数要小，那么可以排除下方一列的所有数都可以排除，点位只能左移。当索引越界时，即返回false。

笔者在梳理好思路进行第一次代码编写时程序提示matrix[0].length报错，在探究后发现错漏了特殊情况，在补上对matrix的判断后运行通过。在构建程序前考虑特殊情况是非常重要的习惯，笔者虽然道行尚浅，但这也不是忽视这步的借口。望共勉。

offer .05替换空格

请实现一个函数，把字符串 s 中的每个空格替换成”%20”。

刚开始被题干中的“替换”抓住了，围绕着替换去做发现并不是很简洁。事实上我们不应该陷入替换这个字眼，我们完全可以创建一个新字符串，让他和替换后的字符串一致。由此笔者想到了StringBuffer这个好东西。

创建一个空StringBuffer—sb，将原字符串与空格一一比较，相等则在sb后append%20，不等则append比较的字符。

做算法应该要尽量屏蔽语言特性，在这里笔者用了StringBuffer，但替换改构造的思想是没有问题的，而事实上不借助StringBuffer构造字符串同样简单，这里就不再修改了。(懒狗笨蛋码农遇不到甜甜的恋爱)

offer 06.从尾到头打印链表

输入一个链表的头节点，从尾到头反过来返回每个节点的值（用数组返回）。

先得到链表长度，然后创建数组，从尾到头填入数据即可。唯一值得一提的是读取链表是从头到尾，而输出数组是从尾到头，这种先入后出的情况可以考虑栈(stack)。

offer 07.重建二叉树

输入某二叉树的前序遍历和中序遍历的结果，请构建该二叉树并返回其根节点。假设输入的前序遍历和中序遍历的结果中都不含重复的数字。

咳咳。。看题解前还真没做出来……这题需要抓住的点是前序遍历、中序遍历的特点，以此寻找整棵二叉树的性质。

对一棵二叉树，前序遍历的结果形式是{根节点[左子树前序遍历结果][右子树前序遍历结果]}，中序遍历的结果形式是{[左子树中序遍历结果]根节点[右子树中序遍历结果]}，由此两种遍历结合可以求出每一个树/子树的根节点，自然而然就想到了递归的做法。每一轮递归中我们根据前序遍历的特点构建根节点，根据中序遍历的特点得到左右子树的元素个数，进而可以分割前序遍历的结果，左右节点由下一轮递归得出。

PS 中序遍历的结果可以以Map<值,索引>的形式存储

offer 09.用两个栈实现队列

用两个栈实现一个队列。队列的声明如下，请实现它的两个函数 appendTail 和 deleteHead ，分别完成在队列尾部插入整数和在队列头部删除整数的功能。(若队列中没有元素，deleteHead 操作返回 -1 )

没什么难度，一个栈控制输入，一个栈控制输出，新元素入队列即为入输入栈，出队列则输出栈输出元素，当输出栈为空时，输入栈将栈内元素全部压入输出栈，重复操作。

唯一的难点可能是部分初学者不会调用语言内置的栈功能。特别注意的是，java的栈官方用的是Deque而非Stack，因为Stack继承自Vector，而Vector存在效率问题，且Deque更为强大，此篇为算法文章便不进行深入了。

offer 10-1.斐波那契数列

写一个函数，输入 n ，求斐波那契（Fibonacci）数列的第 n 项（即 F(N)）。斐波那契数列的定义如下：

F(0) = 0, F(1) = 1
F(N) = F(N - 1) + F(N - 2), 其中 N > 1.

斐波那契数列由 0 和 1 开始，之后的斐波那契数就是由之前的两数相加而得出。答案需要取模 1e9+7（1000000007），如计算初始结果为：1000000008，请返回 1。

DP经典中的经典题了，相信大部分码农接触的第一道DP就是这个，不多赘述了。

offer 10-2.青蛙跳台问题

一只青蛙一次可以跳上1级台阶，也可以跳上2级台阶。求该青蛙跳上一个 n 级的台阶总共有多少种跳法。

答案需要取模 1e9+7（1000000007），如计算初始结果为：1000000008，请返回 1。

本质上就是斐波那契数列问题。对于任意n大于2的台阶，我们可以只看青蛙最后一次跳跃，只有两种可能，要么跳一级，要么二级。跳一级的情况，跳法最大数就是跳n-1级台阶的跳法数，同理最后跳二级的跳法数就是跳n-2级的跳法数，两者相加就是跳n级台阶的跳法数。

offer 11.旋转数组的最小数字

把一个数组最开始的若干个元素搬到数组的末尾，我们称之为数组的旋转。

给你一个可能存在 重复 元素值的数组 numbers ，它原来是一个升序排列的数组，并按上述情形进行了一次旋转。请返回旋转数组的最小元素。例如，数组 [3,4,5,1,2] 为 [1,2,3,4,5] 的一次旋转，该数组的最小值为 1。

本题采用二分思想优化效率。对于旋转后的数组的末值，我们简单的分析就可以得出，在最小值左边的所有值都不会比末值小，在最小值右边的都不会比末值大，用二分查找可分为三种情况：

1、numbers[right] > numbers[mid]

这种情况可以确定mid的右边不存在最小值，而mid本身是不是最小值不得而知，因此将right覆盖为mid。

2、numbers[right] < numbers[mid]

mid的左边以及mid本身一定比right大，都可以排除，因此将left覆盖为mid+1。

3、numbers[right] = numbers[mid]

我们无法得知mid具体情况，只能知道right存在至少一个替代值，因此我们将right左移一位。

迭代到最后，剩下的那个结果就是最终所求

offer 12.矩阵中的路径

给定一个 m x n 二维字符网格 board 和一个字符串单词 word 。如果 word 存在于网格中，返回 true ；否则，返回 false 。

单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母不允许被重复使用。

其实就是一道DFS，暴力回溯解就行，但是要注意递归调用后要恢复visited[][]数组，否则会影响其他方向的递归。

offer 13.机器人的运动范围

地上有一个m行n列的方格，从坐标 [0,0] 到坐标 [m-1,n-1] 。一个机器人从坐标 [0, 0] 的格子开始移动，它每次可以向左、右、上、下移动一格（不能移动到方格外），也不能进入行坐标和列坐标的数位之和大于k的格子。例如，当k为18时，机器人能够进入方格 [35, 37] ，因为3+5+3+7=18。但它不能进入方格 [35, 38]，因为3+5+3+8=19。请问该机器人能够到达多少个格子？

很容易就能想到用深度优先或者广度优先搜索，这里我们用深度优先去解。而限制方格在观察后可以得出搜索只需考虑向下与向左的结论

offer 14-1.剪绳子

给你一根长度为 n 的绳子，请把绳子剪成整数长度的 m 段（m、n都是整数，n>1并且m>1），每段绳子的长度记为 k[0],k[1]…k[m-1] 。请问 k[0]k[1]*…*k[m-1] 可能的最大乘积是多少？例如，当绳子的长度是8时，我们把它剪成长度分别为2、3、3的三段，此时得到的最大乘积是18。

笔者的解法涉及一点数学证明。

1、由奇偶关系，任意一个大于2的数都可以由若干个2与3相加而成

2、2 * 2 > 1 * 3, 3 * 3 > 2 * 2 * 2 > 1 * 2 * 3 > 1 * 5, 2 * 3 > 1 * 4,所以将数拆成2与3的组合乘积最大（3要尽可能多）

因此可以考虑DP

3要尽可能多，因此我们也可以考虑贪心法，不考虑过短情况对一段绳子先全切成长度为3的小段，那么最后剩下的长度只可能为0、1、2

剩下长度为0时，直接将所有小段长度相乘即可。

剩下长度为1时，我们将一个3与1结合为4，再拆分为2*2。

剩余长度为2时，全部相乘。

offer 15.二进制中1的个数

编写一个函数，输入是一个无符号整数（以二进制串的形式），返回其二进制表达式中数字位数为 ‘1’ 的个数（也被称为 汉明重量）。

任何数&0都等于0，因此我们要判断哪位是不是1，只要将该数与相应位为最高位且只有该位定为1的数做&运算即可。如10110000，我们要判断从左往右第三位是不是1，则将10110000与100000做&运算，若结果为0则第三位不是1，若不为零则是1。遍历一遍即可得到最终结果。

优化

n&(n-1)的结果是n最后一个1翻转为0，例如10110000&(10110000-1)=10100000，因此只要计算n与n-1做多少次&运算即可得到结果。

offer 19.正则表达式匹配

请实现一个函数用来匹配包含’. ‘和’‘的正则表达式。模式中的字符’.’表示任意一个字符，而’‘表示它前面的字符可以出现任意次（含0次）。在本题中，匹配是指字符串的所有字符匹配整个模式。例如，字符串”aaa”与模式”a.a”和”ab*ac*a”匹配，但与”aa.a”和”ab*a”均不匹配。

其实。。不容易做吧。

本题使用动态规划，创建一个二维dp数组proc，存储字符串s与正则表达式p的阶段匹配结果，如proc[m][n]的意思是s的前m位与p的前n位是否匹配(proc[0][0]是sp都空时)。

考虑好dp数组的含义后，接下来就是探究dp元素间的关系了。我们将匹配过程分为两大类。

1、p[n] != ‘*’时

这种情况很简单，若p[n] == s[m]，那么proc[m][n]就与进行这次匹配前一个状态相同，即proc[m][n] = proc[m-1][n-1]，若p[n] != s[m]那proc[m][n]就为false。

*2、p[n] == '‘时 **

这个就有点麻烦了，我们要分成两大种情况。’*‘起的作用是量化前一位字符，我们将*与他前一位字符看成一个整体

*前数与s[m]相等时

①当c*代表null时

相当于在正则匹配时将c*这两个字符删去了，即proc[m][n] = proc[m][n-2]

②当c*代表c时

相当于删去了*，即proc[m][n] = proc[m][n-1]

③当c*代表ccc……时

*复制前数多次的话，如果匹配是成功的，那么s[m]也会复制这么一串，那么p[n]与s[m-1]匹配也应没有问题，反之则错误，因此proc[m][n] = proc[m-1][n]。

三种小情况只要成功一组即为匹配成功

*前数与p[n]不等时

只存在上述①号情况，只可能proc[m][n] = proc[m][n-2]
